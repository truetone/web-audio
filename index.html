Generative Music (Experiment 1)

December 24, 2017

This experiment is inspired by Brian Eno's concept of generative music, which is:

any music that is ever-different & changing, created by a system.

These slides inspired me to think about looping in the context of generative music, which then inspired me to start exploring the JavaScript web audio API again. This experiment is the first step in that direction.

I'm creating several oscillator tones with varying wave forms (sine, square, triangle and sawtooth). A couple sets of tones are configured to play a C major chord in the background. Another set of four tones plays the same simple melody in a loop, but at varying times. Finally, there is a set of bass tones that play in regular time to create a predictable rhythm.

The notes of the melody change over time, go in and out of phase and become more dissonant the longer you listen. Eventually the notes resolve back into a more consonant chord.

In my opinion, the longer you listen, the more interesting this gets. Occasionally, the tones create some really interesting harmonics. None of this is recorded. This is all being created in real time, by the browser.

I also used this opportunity to play around with the CSS Grid for layout.

This first experiment won't work yet on iOS devices because Apple puts AudioContext into a suspended state that requires user interaction to resume. At this point I don't have anything set up to do that. This is experimental after all. Also this may break at any time since the API is still changing.
